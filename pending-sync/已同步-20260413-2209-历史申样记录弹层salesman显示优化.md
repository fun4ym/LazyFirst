# 历史申样记录弹层salesman显示优化

## 修改时间
2026-04-13 22:09

## 修改内容
在样品管理的"历史申样记录"弹层中优化了salesman的显示方式，根据主人要求：
1. salesman直接展示到"审批状态"后面
2. 不用展示"BD"多余文本
3. 不用换行

### 修改的文件
1. `frontend/src/views/samples/Management.vue`
   - 修改了第165-176行的历史申样记录弹层布局
   - 移除了独立的BD显示单元格
   - 将salesman直接放在审批状态的el-tag后面
   - 添加了`.salesman-text`CSS样式

2. `frontend/src/views/samples/ManagementBDSelf.vue`
   - 修改了第117-133行的历史申样记录弹层布局
   - 移除了独立的BD显示单元格
   - 将salesman直接放在审批状态的el-tag后面
   - 添加了`.salesman-text`CSS样式

### 修改后的代码示例
```html
<div class="submission-cell">
  <span class="cell-label">审批状态：</span>
  <span class="cell-value">
    <el-tag :type="getSampleStatusType(sub.sampleStatus)" size="small" class="status-tag">
      {{ getSampleStatusText(sub.sampleStatus) }}
    </el-tag>
    <span class="salesman-text">{{ sub.salesman || '' }}</span>
  </span>
</div>
```

### 新增的CSS样式
```css
.salesman-text {
  margin-left: 8px;
  color: #666;
  font-size: 12px;
  vertical-align: middle;
}
```

## 功能说明
- 在样品管理和BD自管理页面中，历史申样记录弹层现在将salesman直接显示在审批状态标签后面
- 移除了"BD："标签，使界面更简洁
- salesman文本使用灰色显示，与审批状态标签形成视觉区分

## 影响范围
- 前端显示修改
- 不影响后端逻辑
- 仅影响/samples和/samples-bd页面的历史申样记录弹层

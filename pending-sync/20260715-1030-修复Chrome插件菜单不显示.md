# 修复 admin 账号看不到 Chrome 插件菜单

**日期**: 2026-07-15
**执行人**: 小垃圾
**状态**: ✅ 已同步

## 根因
前端 `auth.js` 超级管理员判断只认 `'超级管理员'` 和 `'admin'`，而后端 `checkPermission.js` 同时认 `'管理员'`。若 admin 角色的 `name` 为 `'管理员'`，后端 API 能过但前端菜单判断失败 → BD 工作台（含 Chrome 插件）隐藏。

## 修改文件
1. `frontend/src/utils/auth.js`: getPermissions/getUserDataScope 添加 `role.name === '管理员'`
2. `frontend/src/layouts/MainLayout.vue`: isSuperAdmin 添加 `'管理员'` 判断
3. `server/routes/auth.js`: 登录接口 roleId 为 null 时回退到旧 role 字符串

## 部署
- 数据库已备份
- 前后端容器已重建（--no-cache）
- HTTP 200 验证通过

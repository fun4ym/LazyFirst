# 视频登记权限配置修复

## 问题描述
在本地 `/settings/roles` 页面，新增/编辑角色时，未见 `/videos` 的菜单和控件权限配置内容。

## 根本原因
`frontend/src/views/settings/Roles.vue` 文件中的菜单树 (`menuTree`) 在 `BD工作台` 下缺少 `videos` 子菜单配置，导致角色编辑界面无法显示视频登记相关的权限配置选项。

## 修复方案
已修改 `Roles.vue` 文件，添加了 `videos` 菜单项及其权限配置：

### 1. 菜单树添加视频登记子菜单
在 `BD工作台 (bdWorkspace)` 的 `children` 数组中添加了 `videos` 节点：
```javascript
{
  name: '视频登记',
  code: 'videos',
  operations: ['read', 'create', 'update', 'delete'],
  controls: [
    { code: 'btn-view', name: '查看详情' },
    { code: 'btn-add', name: '新增视频' },
    { code: 'btn-edit', name: '编辑视频' },
    { code: 'btn-delete', name: '删除视频' }
  ]
}
```

### 2. 添加菜单名称翻译映射
在 `menuNameMap` 计算属性中添加了 `videos` 的翻译映射：
```javascript
videos: t('menu.videoRegister')
```

### 3. 权限系统一致性验证
- ✅ 前端路由 `/videos` 已在 `permissions.js` 的 `PERMISSION_MAP` 中配置
- ✅ 后端API权限检查使用 `videos:read`、`videos:create`、`videos:update`、`videos:delete`
- ✅ 数据权限配置已在 `MODULE_DATA_SCOPE` 中包含 `videos` 模块
- ✅ 主布局菜单权限检查已包含 `videoRegister()` 函数

## 技术细节
- **操作权限**：`read` (查看), `create` (新增), `update` (编辑), `delete` (删除)
- **控件权限**：查看详情、新增视频、编辑视频、删除视频
- **数据权限**：基于 `createdBy` 字段的数据范围控制
- **权限映射**：控件权限自动映射到对应的操作权限（通过 `controlToOperationMap`）

## 影响范围
1. **角色管理页面**：现在可以配置视频登记模块的菜单权限、控件权限和数据权限
2. **菜单显示**：拥有 `videos:read` 权限的用户可以看到视频登记菜单项
3. **权限检查**：前端路由守卫和后端API均按配置的权限进行验证

## 验证方法
1. 访问 `http://localhost:3000/settings/roles`
2. 创建或编辑角色，在BD工作台下应能看到"视频登记"菜单项
3. 选中"视频登记"菜单，右侧应显示4个控件权限开关
4. 可以配置该模块的数据权限（只看自己、看本部门、看全部）

## 文件修改清单
1. `frontend/src/views/settings/Roles.vue`
   - 第319-349行：在 `bdWorkspace.children` 数组中添加 `videos` 节点
   - 第211-236行：在 `menuNameMap` 中添加 `videos` 翻译映射

## 备注
- 控件权限使用通用控件代码（btn-view, btn-add, btn-edit, btn-delete）
- 控件名称显示为通用翻译（"查看详情"、"新增"、"编辑"、"删除"），但结合菜单名称可明确上下文
- 如需更精确的控件名称，可考虑添加模块特定的控件代码和翻译

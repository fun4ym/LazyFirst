# 修复快递单号提交逻辑

## 修改时间
2026-04-15 15:20

## 问题描述
之前只移除了前端表单的 `required` 属性，但忘记修改提交逻辑，导致即使不填快递单号，空值也被强制提交到后端。

## 修改内容
修复样品管理功能中"快递单号非必填"的提交逻辑问题。

## 修改文件
1. `frontend/src/views/samples/Management.vue`
   - 第1618-1625行，修改 `handleSampleStatusSave` 函数
   - 只有当 `logisticsCompany` 和 `trackingNumber` 有值时才提交到后端
   - 避免空值覆盖数据库中的原有数据

## 修改前代码
```javascript
// 已寄样时发送物流信息
if (sampleStatusForm.sampleStatus === 'sent') {
  payload.logisticsCompany = sampleStatusForm.logisticsCompany
  payload.trackingNumber = sampleStatusForm.trackingNumber
}
```

## 修改后代码
```javascript
// 已寄样时发送物流信息（快递单号非必填）
if (sampleStatusForm.sampleStatus === 'sent') {
  if (sampleStatusForm.logisticsCompany) {
    payload.logisticsCompany = sampleStatusForm.logisticsCompany
  }
  if (sampleStatusForm.trackingNumber) {
    payload.trackingNumber = sampleStatusForm.trackingNumber
  }
}
```

## 修改原因
1. 快递单号应为非必填项，但原逻辑会强制提交空值
2. 空值会覆盖数据库中的原有数据
3. 正确的逻辑应该是：有值才提交，无值则不更新该字段

## 功能影响
- 寄样状态选择"已寄样"时，快递单号和物流公司变为真正的非必填
- 只有填写的字段才会更新到数据库
- 不填写时不会覆盖数据库中的原有数据

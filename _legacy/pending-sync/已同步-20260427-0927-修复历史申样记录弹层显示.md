# 修复历史申样记录弹层显示

## 修改时间
2026-04-27 09:27（更新于09:29）

## 修复的问题
1. `/samples` 页面中，鼠标移入 `Influencer` 列出现的“历史申样记录”弹层：
   - “商品名称”和“TikTok ID”字段未显示内容
   - 申请记录展示格式不符合要求
   - `salesman` 字段未显示

**根本原因分析：**
- 数据库中的 `previousSubmissions` 数组存储的 `salesman` 字段是 `salesmanId`（ObjectId），不是用户名
- 旧数据缺失 `productName`、`influencerAccount`、`sampleStatus`、`salesman` 等字段
- 前端显示逻辑只检查 `sub.salesmanId || sub.salesman`，当两者都为空时不显示任何内容

## 修改的文件

### 1. 后端修复：`server/routes/samples.js`
**两处关键修改：**

#### (1) 创建样品时（第463-471行）
- 添加逻辑：查询 `User` 集合，将 `salesmanId` 转换为用户名
- 构建 `salesmanMap` 映射表，批量查询提高效率
- 存储 `salesman` 字段为用户名（若查询失败则保留原始ID）

**修改后代码：**
```javascript
// 获取所有不重复的salesmanId
const salesmanIds = [...new Set(previousRecords.map(r => r.salesmanId).filter(id => id))];
const salesmanMap = new Map();
if (salesmanIds.length > 0) {
  const users = await User.find({ _id: { $in: salesmanIds } }).select('username _id');
  users.forEach(user => {
    salesmanMap.set(user._id.toString(), user.username);
  });
}
const previousSubmissions = previousRecords.map(record => ({
  sampleId: record._id,
  date: record.date,
  productName: product.name,
  influencerAccount: influencer.tiktokId,
  sampleStatus: record.sampleStatus,
  salesman: record.salesmanId ? (salesmanMap.get(record.salesmanId.toString()) || record.salesmanId) : '',
  createdAt: record.createdAt
}));
```

#### (2) 导入样品时（第1156-1161行）
- 应用相同的转换逻辑
- 确保导入数据时也存储用户名而非ID

### 2. 前端修复：`frontend/src/views/samples/Management.vue` 和 `ManagementBDSelf.vue`

#### (1) 商品名称和TikTok ID显示
- **问题**：弹层顶部的“商品名称”和“TikTok ID”字段未显示内容
- **修复**：添加回退逻辑，优先从历史记录 `previousSubmissions[0]` 读取，若无则使用当前行 `row` 的数据

**修改后：**
```html
<span class="summary-value">{{ row.previousSubmissions[0].productName || row.productName || '-' }}</span>
<span class="summary-value">{{ row.previousSubmissions[0].influencerAccount || row.influencerAccount || '-' }}</span>
```

#### (2) 申请记录格式调整
- **问题**：原有格式为“申请日期：{date} 审批状态：{status}”两列布局
- **修复**：改为紧凑单行格式“{date} {status} {salesman}”
- **示例**：“申请日期：2026-04-16 审批状态：待审核” → “2026-04-16 待审核 diana”

**修改后布局：**
```html
<div class="submission-row" style="display: flex; align-items: center; gap: 8px;">
  <span>{{ formatDate(sub.date) }}</span>
  <el-tag :type="getSampleStatusType(sub.sampleStatus)" size="small" class="status-tag">
    {{ getSampleStatusText(sub.sampleStatus) }}
  </el-tag>
  <span>{{ sub.salesman || sub.salesmanId || row.salesman || '--' }}</span>
</div>
```

#### (3) salesman字段显示逻辑优化
- **第一层**：显示 `sub.salesman`（后端转换后的用户名）
- **第二层**：显示 `sub.salesmanId`（原始ID，当用户名查询失败时）
- **第三层**：显示 `row.salesman`（当前行的salesman，通常是populate后的用户名）
- **兜底**：显示 `--`

### 3. 数据库现状检查
通过查询发现，现有数据中的 `previousSubmissions` 缺失关键字段：
```javascript
{
  sampleId: ObjectId('...'),
  date: ISODate('...'),
  createdAt: ISODate('...'),
  // 缺失 productName, influencerAccount, sampleStatus, salesman
}
```
**影响**：旧数据仍无法显示完整信息，但新创建/导入的数据将包含完整字段。

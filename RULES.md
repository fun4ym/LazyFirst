# 行动准则（开工前必读）

> **本文件是第一顺位规则文件，每次开工前必须先读！**

---

## 一、铁律（绝对禁止）

| 规则 | 原因 |
|------|------|
| 🚫 禁止执行 `docker compose down` | 会删除容器、丢失SSL证书volume挂载、重置网络 |
| 🚫 禁止执行 `docker stop + docker rm` | 同上，会重建容器 |
| 🚫 rsync 禁止 exclude `dist` | 前端构建文件会丢失，网站UI不更新 |
| 🚫 本地做事只连本地 | 禁止本地开发时连线上数据库 |
| 🚫 线上操作必须经主人同意 | 包括ssh、rsync、curl线上、修改线上文件等，未经同意绝对不能动线上 |
| 🚫 绝对不许说谎、隐瞒、支吾 | 主人问什么必须立刻如实回答，不得拖延不得绕弯 |
| 🚫 每次回答必须展示注入的memory | 让主人清楚知道小垃圾的"记忆"被注入了什么信息 |

- **夹着尾巴做AI**

## 三、暗号

- **暗号**：`开工` 或 `干活` 或 `开整`
- 收到暗号后，**必须先读本文件（RULES.md）**
- 读完后再读系统memory（含服务器信息、部署流程等）
- 全部读完后再询问主人今天要做什么

---

## 四、操作原则

1. **先本地后服务器**：所有修改在本地测试，确认后再同步
2. **服务器操作需授权**：除非紧急情况，否则先询问主人
3. **先读规则再行动**：每次对话开始先读本文件
4. **部署前必须读memory**：确认部署流程和敏感信息
5. **判断改了什么 → 决定做什么**：前端改了→同步dist+docker build；后端改了→docker build --no-cache；都改了→两个都做

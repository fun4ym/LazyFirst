## 用户需求

优化样品管理页面的"履约视频"列和相关弹层，添加复制图标、加热/待加热状态图标、新增视频功能，提升用户体验。

## 功能需求

1. **"履约视频"列改动**：

- 有视频有推流码的记录：显示"点开视频"链接 + 复制图标 + 加热火焰图标(橙色) + 编辑图标
- 有视频无推流码的记录：显示"点开视频"链接 + 复制图标 + 待加热火焰图标(灰色) + 编辑图标
- 新增视频按钮

2. **履约内容弹层改动**：

- 弹层标题：履约内容
- 视频链接输入框（必填）
- 推流码输入框
- 取消/保存按钮

3. **图标替换**：

- 复制图标：使用用户提供的SVG
- 加热图标（橙色火焰）：使用用户提供的SVG
- 待加热图标（灰色火焰）：使用用户提供的SVG
- 新增视频图标：使用用户提供的SVG

4. **补充说明**：

- 每条视频-推流码是一行，有几条视频就列几行
- 该列原功能弹层都去掉，用新的控件和弹层
- 去掉滑块的开关，不再单独展示是否投流
- 以有没有投流码为投流标准：投流码不为空=投流，投流码为空=未投流

## 技术方案

### 技术栈

- Vue 3 + Element Plus
- 自定义SVG图标内联使用
- 剪贴板API实现复制功能

### 实现策略

1. **列渲染优化**：修改`adPromotion`列的template，根据`videoLink`和`videoStreamCode`状态渲染不同图标组合
2. **弹层功能增强**：在`fulfillmentDialog`中增加推流码输入框，合并视频链接和推流码编辑功能
3. **图标集成**：将用户提供的SVG图标作为内联组件或直接使用`<svg>`标签嵌入
4. **状态管理**：保持现有数据结构，通过`isAdPromotion`或`videoStreamCode`判断加热状态

### 实现细节

- 复制功能使用`navigator.clipboard.writeText` API
- 图标样式统一使用16x16px，hover效果
- 保持与现有Element Plus组件风格一致
- 加热状态通过推流码是否存在来判断

## 产品概述

统一前端展示模式与功能优化计划，旨在提升系统组件复用性、视觉一致性和用户体验，同时改进BD匹配逻辑和数据展示完整性。

## 核心功能

### 1. 统一前端展示组件

- **商品信息展示组件**：统一/samples "商品信息"列、/products "TikTok商品"列、/orders "商品"列的展示样式
- **达人信息展示组件**：统一/orders "TikTok ID"列、/samples "TikTok ID"列、/samples/public "达人信息"列的展示样式
- **其他可复用组件**：一键复制按钮、状态图标、商店SVG图标、表格列配置模板
- **组件特性**：
- TikTokID显示 + 一键复制icon
- 第二行显示GMV、粉丝数、月均播、月销量（达人信息）
- 统一视觉样式和交互体验

### 2. /samples/public页面增强

- 增加"商家视角"页签，展示产品相关的视频信息、投流码、达人信息
- 方便商家操作和查看产品推广情况

### 3. BD匹配逻辑优化

- 将/orders的BD匹配从"通过申样匹配BD"改为"订单和视频匹配出BD"
- 提高BD匹配的准确性和实用性

### 4. /videos列表完善

- 添加influencer列，显示达人信息
- 完善视频登记信息的完整性

## 技术栈

- **前端框架**：Vue 3 + Composition API
- **UI组件库**：Element Plus
- **构建工具**：Vite
- **样式方案**：CSS + 现有common.css样式复用
- **后端框架**：Node.js + Express
- **数据库**：MongoDB + Mongoose
- **开发语言**：JavaScript/TypeScript

## 实现方案

### 1. 组件化设计

#### ProductCell组件

- **位置**：`frontend/src/components/ProductCell.vue`
- **props**：`product`对象，包含id, name, tiktokProductId, images, shopName等字段
- **功能**：商品图片预览、TikTokID显示与复制、商品名称、店铺信息
- **样式**：复用现有`.product-cell`、`.product-thumb`、`.product-info`等样式类
- **交互**：图片预览、复制TikTokID、点击商品名称跳转

#### InfluencerCell组件

- **位置**：`frontend/src/components/InfluencerCell.vue`
- **props**：`influencer`对象，包含tiktokId, tiktokName, latestFollowers, latestGmv, monthlySalesCount, avgVideoViews等字段
- **功能**：TikTokID显示与复制、达人昵称、GMV/粉丝数/月均播/月销量统计展示
- **样式**：统一达人信息展示样式，与现有`.influencer-data`样式兼容

#### 其他复用组件

- **CopyButton组件**：`frontend/src/components/CopyButton.vue` - 封装复制功能，支持成功/失败提示
- **StatusIcon组件**：`frontend/src/components/StatusIcon.vue` - 根据状态显示对应图标（完成✅、进行中⏰等）
- **ShopIcon组件**：`frontend/src/components/ShopIcon.vue` - 商店SVG图标组件，替换硬编码的SVG
- **TableColumn组件**：`frontend/src/components/TableColumn.vue` - 可配置的表格列模板（含排序、溢出提示等配置）

### 2. 页面重构计划

#### 需要修改的页面：

1. `frontend/src/views/samples/Management.vue` - 商品信息列、达人信息列替换为组件
2. `frontend/src/views/samples/ManagementBDSelf.vue` - 商品信息列替换为组件
3. `frontend/src/views/samples/PublicCollection.vue` - 商品信息列、达人信息列替换为组件，添加"商家视角"页签
4. `frontend/src/views/report-orders/Index.vue` - 商品列、TikTok ID列替换为组件
5. `frontend/src/views/products/ProductTab.vue` - TikTok商品列替换为组件
6. `frontend/src/views/samples/VideoManagement.vue` - 添加influencer列（使用InfluencerCell组件）

### 3. 数据结构优化

#### 后端API调整：

- 确保`/api/influencers`等相关接口返回influencer完整字段：`latestGmv`, `monthlySalesCount`, `avgVideoViews`
- 视频接口`/api/videos`需要返回关联的达人信息供influencer列显示
- 保持向后兼容，避免影响现有功能

#### BD匹配逻辑重写：

- **当前逻辑**：根据`productId + influencerUsername`匹配样品记录
- **新逻辑**：根据`订单 + 视频`匹配出BD（通过视频记录关联达人，再关联到BD）
- **修改文件**：`server/routes/report-orders.js`中的match-bd接口

### 4. 实现细节

#### 性能考虑：

- 组件复用减少重复代码，降低维护成本
- 保持现有样式类不变，避免样式冲突
- 复制功能使用Clipboard API，提供用户友好的反馈

#### 兼容性：

- 所有组件保持与现有数据结构的兼容性
- 逐步替换，不影响现有功能
- 新组件支持可选的props，处理缺失数据情况

## 架构设计

### 组件层级结构

```
App
├── 页面组件 (Page Components)
│   ├── Management.vue
│   ├── ProductTab.vue
│   └── ...
└── 通用组件 (Common Components)
    ├── ProductCell.vue
    ├── InfluencerCell.vue
    ├── CopyButton.vue
    ├── StatusIcon.vue
    ├── ShopIcon.vue
    └── TableColumn.vue
```

### 数据流向

1. 页面组件通过props传递数据给展示组件
2. 展示组件处理UI展示和用户交互
3. 用户交互（如复制）触发事件，页面组件处理业务逻辑
4. 后端API提供完整的数据支持

## 产品概述

手机端独立公开页面，面向达人展示所有启用商品（active + 有活动链接），无需登录即可访问。目标是让达人浏览商品、看到佣金优势，愿意点击链接帮忙带货。

## 核心功能

- **商品浏览**：展示所有启用商品的图片、名称、TikTok商品ID、类目、达人收益率 vs 广场佣金率
- **搜索筛选**：按类目（下拉，从productCategory去重获取）、店铺（下拉，从Shop获取active店铺）、商品名（模糊）、商品ID（模糊）四维搜索
- **佣金对比**：突出展示达人收益率 vs 广场佣金率的差异，高佣金商品更醒目
- **带货入口**：每个商品提供TikTok活动链接按钮，引导达人点击上架
- **配色方案**：主色 #775999，辅助色 #6DAD19（用于高亮佣金优势、CTA按钮等），白底黑字保证可读性

## 技术栈

- 前端：Vue 3 + Element Plus（与现有项目一致）
- 后端：Express.js + MongoDB（与现有项目一致）
- 样式：Scoped CSS（与PublicRecruitment.vue一致）

## 实现方案

### 后端：新建公开商品API

创建 `server/routes/public-products.js`，挂载 `/api/public/products`，无需认证。

**API设计**：

1. `GET /api/public/products` — 获取启用商品列表（分页+搜索）

- 查询条件：`status: 'active'`，且 `activityConfigs.activityLink` 非空
- 搜索参数：`keyword`（名称/tiktokProductId模糊）、`category`（productCategory精确匹配）、`shopId`（店铺ID）
- populate `shopId` 获取店铺名称
- 返回字段：`_id, name, tiktokProductId, productCategory, images, price, sellingPrice, currency, squareCommissionRate, activityConfigs, shopId.shopName`
- 分页：`page`, `limit`

2. `GET /api/public/products/filters` — 获取筛选项

- 返回所有不重复的 `productCategory` 列表
- 返回所有active的店铺 `{ _id, shopName }`

### 前端：新建公开商品浏览页

创建 `frontend/src/views/products/PublicProducts.vue`，路由 `/products/public`，`requiresAuth: false`。

**页面结构**（从上到下）：

1. 顶部色块（#775999）：品牌信息 + Logo
2. 搜索栏：类目下拉 + 店铺下拉 + 关键词输入
3. 商品瀑布流/列表：每个商品卡片展示图片、名称、ID、类目、佣金对比
4. 底部色块：版权信息

**佣金对比展示逻辑**：

- 达人收益率 = activityConfigs中默认活动的 promotionInfluencerRate，无则用 commissionRate
- 广场佣金率 = squareCommissionRate
- 差异 = 达人收益率 - 广场佣金率，差异 > 0 时用 #6DAD19 绿色高亮

### 路由注册

- 后端：在 server.js 中 `app.use('/api/public/products', publicProductsRoutes)`
- 前端：在 router/index.js 中添加 `/products/public` 路由，`requiresAuth: false`

## 实现要点

- 商品只展示 `status: 'active'` 且有活动链接的，避免无链接商品
- productCategory 是自由文本，用 `distinct()` 去重获取筛选项
- 店铺列表从 Shop 模型获取，只取 `status: 'active'`
- 佣金率存储为小数（0.08 = 8%），前端展示时乘100加%
- 参考 PublicRecruitment.vue 的页面结构和样式模式

## 推广追踪：统计哪个系统用户引导达人访问

### 方案：推广链接带ref参数

系统用户（BD）在系统内复制推广链接时，链接自动带上自己的标识：

```
https://shop.lazyfirst.com/products/public?ref=U57f8a3b2
```

### 实现层级

| 层级 | 操作 |
| --- | --- |
| **系统内** | 在"产品管理"或"招募"页面添加"复制推广链接"按钮，点击后生成带 `ref=用户ID前8位` 的链接 |
| **公开页前端** | 页面加载时读取 `route.query.ref`，存入 localStorage，后续所有API请求都带上 `X-Ref-User` header |
| **后端API** | `/api/public/products` 接口读取 `X-Ref-User` header，写入 PageVisit 日志集合 |
| **数据统计** | 后期新增 `GET /api/stats/referrals` 接口，统计每个BD引导了多少达人访问、点击了多少商品 |


### 新增模型

```js
// server/models/PageVisit.js
{
  userId: String,        // ref参数（BD用户ID前8位）
  page: String,          // 'public-products'
  productId: String,     // 点击的商品ID（可选）
  action: String,        // 'view' | 'click_product' | 'click_tiktok'
  ip: String,
  userAgent: String,
  createdAt: Date
}
```

## 独立站部署：让页面看起来像独立网站

### 当前问题

公开页和系统管理页在同一个Vue SPA中，URL是 `tap.lazyfirst.com/products/public`，明显是系统子页面。

### 方案：独立子域名 + 同一前端构建

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| **DNS** | 新增 `shop.lazyfirst.com` A记录指向同一台服务器 | 子域名完全不同于 `tap.lazyfirst.com` |
| **Nginx** | 新增 `server` 块，`server_name shop.lazyfirst.com`，指向同一份前端静态文件 | 同一IP，不同域名 |
| **前端路由** | PublicProducts.vue 不使用 MainLayout，独立渲染，有自己的顶部导航和Footer | 达人看不到后台系统任何痕迹 |
| **SEO** | 添加 `<title>`、`<meta description>`、Open Graph 标签 | 分享到社交媒体时显示商品图片和描述 |


### Nginx配置新增（部署时加）

```
server {
    listen 80;
    server_name shop.lazyfirst.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name shop.lazyfirst.com;

    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 关键点

- `shop.lazyfirst.com` 和 `tap.lazyfirst.com` 共享同一份前端构建产物，通过路由和host判断展示形式
- 达人访问 `shop.lazyfirst.com` 时只看到商品展示页，完全看不到后台系统
- 部署时需要：DNS解析 + SSL证书（可用通配符证书 `*.lazyfirst.com`）+ Nginx配置

# LazyFirst/TAP系统 - 项目架构与市场分析报告

## 一、项目概述

**LazyFirst TAP系统** 是一个面向TikTok电商生态的多租户SaaS平台，专为达人管理、样品申请、订单追踪和佣金分润设计的全链路解决方案。

**核心价值定位**：
- **达人资源管理**：建立系统化的TikTok达人数据库与CRM
- **样品流程优化**：从申样、寄样到履约追踪的全流程自动化
- **佣金分润透明化**：标准化的GMV追踪与分润结算体系
- **数据驱动决策**：基于达人画像和销售数据的智能匹配

### 2.2 核心数据模型设计（24个集合）

#### 2.2.1 系统基础模型
1. **User** - 用户系统（多租户隔离，角色权限，密码加密）
2. **Company** - 公司管理（多租户基础，系统配置）
3. **Role** - 角色权限（细粒度权限控制，数据范围）
4. **Department** - 部门管理（树形结构，部门负责人）

#### 2.2.2 达人管理模型
5. **Influencer** - TikTok达人管理
   - 公海/私海池机制
   - 自动维护状态计算（7/14/21/24天梯度）
   - 达人画像：粉丝数、GMV、月销件数、视频均播
   - 黑名单管理
6. **InfluencerMaintenance** - 达人维护记录

#### 2.2.3 商品供应链模型
7. **Product** - 商品管理
   - 多活动配置（activityConfigs数组）
   - 佣金设置（推广/广告双模式）
   - 样品要求（线上/线下模式）
   - 店铺关联
8. **Shop系列模型** - 店铺管理系统（Shop/ShopContact/ShopRating/ShopTracking）

#### 2.2.4 业务流程模型
9. **SampleManagement** - 样品管理
   - 申请 → 寄样 → 收货 → 履约 → 出单完整闭环
   - 状态追踪（pending/shipping/sent/refused）
10. **Order** - 订单管理（原生TikTok订单数据结构）
11. **Commission** - 佣金记录
12. **Recruitment** - 招募配置（公开招募页面，支持市场投放）

#### 2.2.5 数据分析模型
13. **ReportOrder** - 报表订单（聚合数据报表）
14. **BdDaily** - BD日报统计
15. **Bill** - 账单管理（佣金结算）
16. **Performance** - 绩效报表
17. **Activity** - TikTok活动管理

## 三、核心业务逻辑模块

### 3.1 达人池管理机制

```
┌─────────────────┐     ┌─────────────────┐
│    公海池       │     │    私海池       │
│    (Public)     │     │   (Private)     │
├─────────────────┤     ├─────────────────┤
│ • 无人认领      │     │ • BD专属        │
│ • 全员可见      │     │ • 需定期维护    │
│ • 可认领分配    │     │ • 自动释放逻辑  │
└─────────────────┘     └─────────────────┘
         ↑                         │
         └────────┬────────────────┘
                  │
          [BD点击"认领"按钮]
```

**自动释放算法**（基于lastMaintenanceTime）：
- 0-7天：✅ **正常**
- 8-14天：⚠️ **需要维护**
- 15-21天：⚠️ **高风险**（即将释放）
- 22-24天：⚠️ **即将释放**
- 25+天：🗑️ **已释放**（自动回公海）

### 3.3 佣金计算体系

1. Affiliate Partner Commission
2. Creator Commission  
3. Service Provider Reward Commission
4. Influencer Reward Commission

6. Influencer Shop Ad Commission

**计算规则**：
```javascript
// 活动级佣金配置
activityConfigs: [{
  promotionInfluencerRate: Number,  // 推广达人佣金率

  promotionCompanyRate: Number,     // 推广公司佣金率
  adInfluencerRate: Number,         // 广告达人佣金率
  adOriginalRate: Number,           // 广告原创佣金率
  adCompanyRate: Number             // 广告公司佣金率
}]

// 结算周期：月结/周结可配置
```

## 四、市场分析与定位

### 4.1 目标市场分析

#### 市场机会点：
1. **TikTok电商爆发期**（2023-2025 CAGR > 50%）
   - 东南亚市场：GMV年增长80%+
   - 泰国市场：TikTok Shop渗透率快速提升
   - 达人经济：商业化需求旺盛

2. **行业痛点**：
   - 达人管理混乱（Excel手工记录）
   - 样品流程不透明（履约率低）
   - 佣金结算复杂（Excel公式易错）
   - 数据孤岛严重（缺乏全局视图）

3. **竞争格局**：
   - **国际大厂**：Salesforce/Shopify（价格高，本土化差）
   - **本土解决方案**：定制化程度低，功能单一
   - **空白市场**：专注于TikTok生态的SaaS平台

### 4.2 用户画像分析

#### 核心用户群体：
1. **品牌方/代理商**（付费主体）
   - 需求：达人管理、样品追踪、ROI分析
   - 痛点：多品牌多Campaign管理混乱
   - 预算：$500-$5,000/月

2. **业务拓展（BD）团队**（日常使用者）
   - 需求：公海抢单、维护提醒、绩效报表
   - 痛点：手动维护达人信息耗时耗力
   - 期望：减少重复劳动，提效工具

3. **TikTok达人**（间接用户）
   - 需求：样品申请、佣金透明、合作机会
   - 痛点：品牌方沟通效率低
   - 期望：一站式合作平台
